## What it is

Meeting Me Memory is a production web app I built to own the full loop from
**"I'm in a call"** to **"I can find that recording later."** It combines a
browser recorder, a tagged cloud library, calendar sync, and a tasks layer,
plus Mya, a small assistant (in-app chat + Chrome extension) that keeps
recording top-of-mind when I'm on Google Meet, Teams, or Zoom.

## How it works

- **Record.** The `/record` page uses the browser's screen-capture APIs to
  record meetings, optionally trim with FFmpeg.wasm, categorise on save, and
  either download locally (free, no account) or upload to cloud storage.
- **Store.** Firestore holds metadata (tags, categories, meeting links); blobs
  land in Firebase Storage, Google Drive, or OneDrive depending on the user's
  auth provider and storage policy.
- **Sync.** OAuth flows for Google, Microsoft, and Zoho pull calendar events
  into Firestore; recordings are matched to meetings by time window.
- **Find.** The library and home dashboard support search/filter by title, tag,
  and category. Tasks can be created from meetings for follow-up work.
- **Assist.** Mya answers scoped help questions via a server-side Claude route;
  the Chrome extension detects call URLs and surfaces reminders without bundling
  auth secrets.

## Why it matters

It's an end-to-end product build, not just a UI mock. OAuth across three
calendar providers, resumable multi-backend uploads, Firebase security rules,
production deploy on Vercel, and a companion extension that bridges the web
app and the meeting tabs I actually use day to day.
